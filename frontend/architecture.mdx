The frontend is a Next.js 16 application deployed on Vercel. It is the only surface voters and admins interact with directly. All cryptographic operations triggered by the voter — blinding a ballot, unblinding a signature, encrypting the final ciphertext — happen in the browser before anything is sent to the server.

## Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 16, App Router |
| Language | TypeScript |
| Styling | Tailwind CSS v4 |

## Four-entity trust model

```mermaid
graph TD
    Voter["Voter (Browser)"]
    Admin["Administrator"]
    Commissioner["Commissioner"]
    Counter["Counter"]

    Voter -->|"N1 eligibility check"| Commissioner
    Voter -->|"blinded ballot"| Admin
    Admin -->|"blind signature s'"| Voter
    Voter -->|"encrypted ciphertext C"| Counter
    Counter -->|"public key"| Voter

    note1["Voter unblinds s' locally\nAdministrator never sees ballot content"]
```

| Entity | Knows | Does not know |
|---|---|---|
| Voter | Their own N1, N2, vote choice, blinding factor k | Other voters' choices |
| Administrator | Voter identity via N1/commissioner check | Vote content (blind signatures) |
| Commissioner | Valid N1 list and N2 hashes | Vote choices or ballot content |
| Counter | Decrypted vote content | Which voter cast which ballot |

## Election status as a state machine

The entire UI branches on one value: `voting_status`. Every page checks it and redirects if the precondition is not met.

```mermaid
stateDiagram-v2
    [*] --> register : election created
    register --> vote_started : admin clicks Start Vote
    vote_started --> vote_ended : admin clicks End Vote

    register : /register active\n/vote blocked
    vote_started : /vote active\nadmin sees participation count
    vote_ended : results published\n/results/verify-vote active
```

## Where in-browser crypto happens

The backend never sees a plaintext ballot. By the time any ballot data leaves the browser it has been blinded, signed, unblinded, and encrypted — in that order.

```mermaid
sequenceDiagram
    participant Browser
    participant Backend

    Browser->>Browser: assemble ballot (vote, N2, random_bits)
    Browser->>Browser: pick random k, compute m' = m * k^e mod N_admin
    Browser->>Backend: POST /voters/check_n1
    Backend-->>Browser: eligibility confirmed
    Browser->>Backend: send blinded ballot m'
    Backend-->>Browser: blind signature s'
    Browser->>Browser: unblind: s = s' * k^-1 mod N_admin
    Browser->>Backend: GET /vote/public-key
    Backend-->>Browser: (e_counter, N_counter)
    Browser->>Browser: encrypt: C = s^e_counter mod N_counter
    Browser->>Backend: POST /voters/submit_vote (C)
    Backend-->>Browser: accepted
```
